# 릴리즈 노트

## 1.0.1 (2026-02-02)

### 공유 링크·다운로드·이력 개선

검증 결과를 링크로 공유하고, 지금까지의 문서에서 바로 이동·복사할 수 있도록 개선했습니다.

#### 검증 결과 공유 링크

- **URL hash에 저장**: 검증할 텍스트/엑셀 결과를 URL hash(`#d=...`)에 저장해 공유·북마크할 수 있습니다.
- **431 방지**: gzip 압축 + base64url 인코딩으로 Request URI Too Long(431)를 방지하고, 같은 URL 길이에 더 많은 데이터를 담습니다.
- **자동 갱신**: 검증하기 또는 엑셀 업로드 후 주소창 URL이 자동으로 갱신되며, 해당 링크를 복사해 공유할 수 있습니다.
- **링크로 접속**: 공유 링크로 접속하면 텍스트가 복원되고 한 번만 자동 검증됩니다.

#### 지금까지의 문서

- **공유 링크 저장**: 지금까지의 문서에 각 검증의 공유 링크가 저장됩니다.
- **클릭 시 이동**: 항목을 클릭하면 해당 문서(검증 결과) 링크로 이동합니다. (같은 페이지에서 hash만 변경되어 새로고침 없이 전환)
- **링크 복사**: 각 항목 옆 「링크 복사」 버튼으로 해당 문서 공유 링크를 클립보드에 복사할 수 있습니다.
- **링크 진입 시 저장**: 링크로 직접 들어온 경우에도 해당 링크가 지금까지의 문서에 저장됩니다.

#### 다운로드·입력 화면

- **검증 결과 다운로드**: 검증 결과 화면에서 「엑셀 다운로드」「텍스트 다운로드」로 현재 검증 데이터를 파일로 저장할 수 있습니다.
- **링크로 저장**: 입력 영역에 「링크로 저장」 버튼이 있어, 검증 전에도 현재 텍스트가 담긴 링크를 복사할 수 있습니다. 검증 결과 화면의 「링크 복사」로도 현재 URL을 복사할 수 있습니다.
- **새로 검증**: 「새로 검증」 클릭 시 URL hash를 비우고 첫 화면(무엇을 검증할까요?)으로 돌아갑니다.

#### 엑셀·파싱 호환

- **엑셀도 링크 공유**: 엑셀(.xlsx, .xls) 업로드 결과도 hash에 저장되어 링크로 공유할 수 있습니다.
- **헤더 인식 개선**: `parseTextData`가 「순번」 단독 줄뿐 아니라 「순번\t거래일자\t...」 탭 구분 헤더 행도 인식합니다. BOM 제거 및 줄바꿈 정규화를 적용했습니다.

#### 호환성

- 예전 쿼리 형식(`?text=...`) 링크는 계속 동작합니다.
- 기존 텍스트 붙여넣기·엑셀 업로드 방식은 그대로 사용할 수 있습니다.
